## Environment Setup

This repo uses [uv](https://docs.astral.sh/uv/) for dependency management:

```bash
# Initial setup (requires submodules)
git submodule update --init --recursive
GIT_LFS_SKIP_SMUDGE=1 uv sync
GIT_LFS_SKIP_SMUDGE=1 uv pip install -e .
```

For PyTorch support, apply transformers patches after sync:
```bash
cp -r ./src/openpi/models_pytorch/transformers_replace/* .venv/lib/python3.11/site-packages/transformers/
```

## Common Commands

```bash
# Run tests
uv run pytest

# Run a single test file
uv run pytest src/openpi/models/model_test.py

# Lint and format
uv run ruff check .
uv run ruff format .

# Run pre-commit (required before PRs)
pre-commit install
pre-commit run --all-files

# Compute norm stats before training (required)
uv run scripts/compute_norm_stats.py --config-name <config_name>

# JAX training
XLA_PYTHON_CLIENT_MEM_FRACTION=0.9 uv run scripts/train.py <config_name> --exp-name=<run_name> --overwrite

# PyTorch training (single GPU)
uv run scripts/train_pytorch.py <config_name> --exp_name <run_name>

# PyTorch training (multi-GPU)
uv run torchrun --standalone --nnodes=1 --nproc_per_node=<num_gpus> scripts/train_pytorch.py <config_name> --exp_name <run_name>

# Serve a trained policy
uv run scripts/serve_policy.py policy:checkpoint --policy.config=<config_name> --policy.dir=<checkpoint_dir>

# Convert JAX checkpoint to PyTorch
uv run examples/convert_jax_model_to_pytorch.py --checkpoint_dir <jax_ckpt> --config_name <config> --output_path <out>
```

## Architecture Overview

### Models

Three model families live under `src/openpi/models/`:
- **Ď€â‚€ / Ď€â‚€.â‚…** (`pi0.py`, `pi0_config.py`): Flow-based vision-language-action (VLA) model using a PaliGemma backbone + SigLIP vision encoder
- **Ď€â‚€-FAST** (`pi0_fast.py`): Autoregressive VLA using the FAST action tokenizer
- **Gemma** (`gemma.py`, `gemma_fast.py`): Language backbone shared by both model families

PyTorch equivalents are in `src/openpi/models_pytorch/` (Ď€â‚€ and Ď€â‚€.â‚… only; no FAST, LoRA, or FSDP support).

The JAX models use Flax NNX. `BaseModelConfig` in `model.py` defines shared fields: `action_dim`, `action_horizon`, `max_token_len`, `model_type`.

### Training Pipeline

`src/openpi/training/config.py` is the central configuration file. It defines:
- `TrainConfig`: top-level training hyperparameters
- `DataConfig` / `DataConfigFactory` subclasses: one per robot platform (ALOHA, DROID, LIBERO, etc.)
- `_CONFIGS`: registry of all named configs (e.g., `pi05_libero`, `pi0_droid`, `debug`)

Use `get_config(name)` to retrieve a config by name. Available named configs include: `pi05_libero`, `pi0_libero`, `pi0_fast_libero`, `pi05_droid`, `pi0_droid`, `pi0_fast_droid`, `pi0_aloha`, `pi05_aloha`, `pi0_aloha_sim`, `debug`, etc.

### Data Pipeline

Data flows through a chain of `DataTransformFn` callables (defined in `src/openpi/transforms.py`). Transforms are grouped into:
1. `repack_transforms`: map raw dataset keys to the common format
2. `data_transforms`: robot-specific pre-normalization transforms
3. `model_transforms`: model-specific transforms (tokenization, image resize, padding)

`DataConfig.norm_stats` holds precomputed normalization statistics; these are generated by `scripts/compute_norm_stats.py` and stored under `./assets/<config_name>/`.

### Policies

`src/openpi/policies/` contains robot-specific `*Inputs` / `*Outputs` classes that map raw sensor observations and actions to/from the model's canonical format. Each robot platform has its own policy file (e.g., `libero_policy.py`, `droid_policy.py`, `aloha_policy.py`). The `Policy` class in `policy.py` wraps a model + transforms for inference.

### Inference & Serving

- `scripts/serve_policy.py`: Starts a websocket policy server on port 8000
- `src/openpi/serving/websocket_policy_server.py`: Server implementation
- `packages/openpi-client/`: Standalone client package for connecting to the policy server
- Checkpoints are auto-downloaded from `gs://openpi-assets` and cached in `~/.cache/openpi` (override with `OPENPI_DATA_HOME`)

### Key Conventions

- All training/data code uses JAX arrays; PyTorch is only used for the `models_pytorch/` path
- Tests live alongside source files as `*_test.py`; tests marked `@pytest.mark.manual` must be run explicitly
- Ruff is the linter/formatter; line length is 120, target Python 3.11
- `third_party/` and `src/openpi/models_pytorch/transformers_replace/` are excluded from linting
